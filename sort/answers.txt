sort1 uses: Bubble Sort

How do you know?: Sort1 uses Bubble sort because here Big O is n squared which is not same as big omega which is n. This is possible only
                  in case of Bubble sort.

sort2 uses: Merge Sort

How do you know?: Sort2 uses Merge sort because it takes least time in all 3 cases. Moreover, it takes approximately same time in worst
                  case and in best case scenario which indiactes this code is for Merge sort.

sort3 uses: Selection Sort

How do you know?: Sort3 uses Selection sort because it takes almost equal time in worst case scenario as compared to Bubble sort which
                  is in order of n squared and also takes approximately same time in best case scenario. This indicates that it is
                  Selection sort.
